## Data and consistency contract

- Search storage is capped at 60 KiB per Entry and generated by Rust, so all supported databases query the same normalized projection without database-specific text-search semantics.
- MySQL 8.4 receives `search_text` through a staged, re-entrant migration: add nullable `TEXT`, backfill bounded 32-row batches, then enforce `NOT NULL`. This avoids the unsupported `TEXT DEFAULT ''` DDL while preserving safe recovery from a partially applied migration.
- Search is Feed-local and uses AND semantics over at most eight canonical terms; the canonical term sequence participates in the user-bound cursor filter hash.
- Bulk mark-read locks the User first and affected Subscriptions in deterministic order, then advances only database-authoritative sparse read frontiers inside one transaction.
- The client confirms and submits the visible snapshot generation; a newer pending snapshot is not marked read accidentally and is promoted only through the normal merge path.
- Unread navigation is derived from the rendered subscription tree and exact unread counts; canonical route URLs remain the navigation source of truth.
