# Raindrop Reader UI v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox syntax for progress.

**Goal:** Build the first real ASTRYX Reader UI over the completed Subscription and Reader APIs.

**Architecture:** Generate Subscription TypeScript contracts from the committed OpenAPI artifact, keep Reader data in one feature-local normalized reducer, and render one responsive AppShell/Layout workspace. API, schema, state, routes, panels, and tests remain separate modules.

**Tech Stack:** React 19, TypeScript 7, React Router 7, ASTRYX 0.1.6, Vitest, Playwright.

## Global Constraints

- No new runtime dependency and no backend migration.
- Subscription DTOs come only from `docs/openapi/subscription-v1.json`.
- Use ASTRYX components before custom markup; no generic control wrappers.
- Feature source files target 250 lines or fewer.
- Desktop >=1100, medium 720-1099, mobile <720; 360x800 and 390x844 have no horizontal overflow.
- High-frequency keyboard actions do not animate; reduced motion is honored.
- Never touch `.superpowers/research/` or `node_modules/`.
